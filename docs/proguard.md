# ProGuard / R8 Configuration for KSPindle

## Why ProGuard/R8 Rules Are Needed

KSPindle relies on several mechanisms that are broken by code obfuscation
unless the correct keep rules are applied:

1. **ServiceLoader discovery** -- `META-INF/services/io.github.oayilix.kspindle.runtime.ServiceIndexProvider`
   files are generated by the KSP processor and read at runtime via `java.util.ServiceLoader`.
   Verify these files are packaged in the final APK/JAR.

2. **Reflective instantiation** -- `Class.getDeclaredConstructor().newInstance()` is used
   by `RegistryEntry` to create service instances. ProGuard renames class and constructor
   names, so `@ServiceProvider`-annotated classes must have their no-arg constructors
   preserved.

3. **KSP-generated implementations** -- The KSP processor generates `ServiceIndexProvider`
   implementations that implement the framework interface. These classes and their
   `initialize(ServiceRegistry)` method must not be renamed or removed.

4. **Public API classes** -- `Kspindle`, `ServiceRegistry`, `ServiceNotFoundException`,
   and `ProviderRegistration` are part of the public API and consumed by user code.

## How the Rules Are Provided

A `consumer-rules.pro` file is located at:

```
kspindle-runtime/consumer-rules.pro
```

It contains all the `-keep` rules listed above, organized by concern with bilingual
comments (English and Chinese).

## How to Apply

### Android Projects

If the KSPindle is published as an Android AAR with `consumerProguardFiles`,
the `consumer-rules.pro` file is automatically merged into the consuming app's
ProGuard/R8 configuration. When consuming the current JVM/JAR module directly,
add the rules manually as shown below.

### JVM Projects (JAR dependency)

For JVM libraries or applications distributed as JARs, manually reference the rules
in your ProGuard/R8 configuration:

**Option A -- In your build.gradle.kts (Android library project):**
```kotlin
android {
    buildTypes {
        release {
            proguardFiles(
                getDefaultProguardFile("proguard-android-optimize.txt"),
                "consumer-rules.pro"   // copy from kspindle-runtime/consumer-rules.pro
            )
        }
    }
}
```

**Option B -- Direct ProGuard include:**
```pro
-include /path/to/kspindle/consumer-rules.pro
```

**Option C -- Copy rules into your own ProGuard configuration:**
Open `kspindle-runtime/consumer-rules.pro` and copy the relevant `-keep` rules into your
project's ProGuard configuration file.

## How to Verify

After building a release artifact (APK, AAR, or JAR), verify that the key classes
are preserved:

### Check META-INF/services is present

```bash
# For APK
unzip -l app-release.apk | grep META-INF/services

# For JAR
jar tf kspindle-runtime-*.jar | grep META-INF/services
```

Expected output:
```
META-INF/services/io.github.oayilix.kspindle.runtime.ServiceIndexProvider
```

### Check ServiceIndexProvider implementations are not obfuscated

```bash
# Dump class names from the release APK
unzip -p app-release.apk classes.dex | strings | grep ServiceIndex
```

If you see `ServiceIndexProvider` and your generated `ServiceIndex_*` classes
with their original names, the rules are working correctly.

### Test at runtime

Run a simple integration test that calls `Kspindle.initialize()` followed by
`Kspindle.load()` with a registered service. If the build succeeds and the
service is returned without `ClassNotFoundException` or
`ServiceNotFoundException`, ProGuard/R8 is configured correctly.

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---|---|---|
| `ServiceNotFoundException` at runtime after release build | `META-INF/services` file missing from the final artifact | Verify Java resources are packaged and inspect the APK/JAR contents |
| `ClassNotFoundException` for `ServiceIndex_*` | KSP-generated class renamed or removed | Verify `-keep class * implements io.github.oayilix.kspindle.runtime.ServiceIndexProvider` |
| `NoSuchMethodException: <init>` | No-arg constructor obfuscated | Verify `-keepclassmembers` rule for `@ServiceProvider` constructors |
